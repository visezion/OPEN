# Security Policy

## Supported Versions

The following versions of this project are currently supported with security updates:

| Version | Supported |
| ------- | --------- |
| 1.x.x   | Yes       |
| < 1.0   | No        |

If you are using an unsupported version, please upgrade to the latest stable release to receive security updates and fixes.

---

## Reporting a Vulnerability

We take security seriously and appreciate responsible disclosure of vulnerabilities.

### How to Report

If you discover a security vulnerability, please report it privately using one of the following methods:

* Email: [security@yourdomain.com](mailto:security@vicezion.com)
* Or open a **private security advisory** on GitHub (preferred)

Do NOT open a public issue for security vulnerabilities.

---

### What to Include

Please include as much detail as possible to help us understand and reproduce the issue:

* Description of the vulnerability
* Steps to reproduce
* Proof of concept (if available)
* Affected components or files
* Potential impact (e.g., data leak, RCE, privilege escalation)

---

### Response Timeline

* Initial response: within 48 hours
* Status update: within 3–5 business days
* Resolution timeline: depends on severity and complexity

---

### What to Expect

After submitting a report:

* Your report will be reviewed and validated
* You may be contacted for additional details
* If confirmed, we will:

  * Work on a fix
  * Release a patch
  * Credit you (if you agree)

If the report is not accepted, we will provide a clear explanation.

---

### Disclosure Policy

* Please do not disclose the vulnerability publicly until a fix has been released
* We will coordinate responsible disclosure where applicable

---

## Security Best Practices

Users are encouraged to:

* Keep dependencies up to date
* Use environment variables for secrets
* Avoid committing sensitive data (API keys, passwords)
* Follow secure deployment practices

---

## Scope

This policy applies to:

* Core application code
* APIs and integrations
* Deployment configurations

Third-party dependencies should be reported to their respective maintainers.

---

Thank you for helping keep this project secure.
